# 代码结构与维护说明

本文档用于快速理解项目代码，避免后续修改时在 `app.py` 中迷路。

## 当前结构

```text
app.py
  Streamlit 主应用。包含登录、上传、解析、摘要、分类、展示、下载和历史记录。

tools/file_router.py
  文件类型识别和任务路由。输入文件名与文本，输出文件类型、任务类别、置信度和推荐输出。

tools/file_router_config.json
  路由配置。新增文件类型、关键词或推荐输出时优先改这里。

tools/prompt_templates.py
  提示词模板与输出规格。摘要风格、语言约束、顶刊/顶会写作合同优先改这里。

academic_abstract_examples.jsonl
  顶刊/顶会摘要风格示例库。只用于提示词风格参考，不作为事实来源。

data/
  运行数据，包括 SQLite 数据库和用户上传文件。不要随意移动或清空。

literature_training_data/
  顶刊/顶会文献采集和训练数据材料。
```

## app.py 代码分区

`app.py` 当前仍是单文件主应用。主要函数按功能分为：

```text
1. 配置与存储
   configure_tesseract
   init_storage
   save_document / list_user_documents / get_user_document

2. 文本质量与乱码处理
   mojibake_marker_count
   repair_mojibake_text
   clean_text
   is_unreadable_text

3. 文献示例与顶刊风格提示
   load_academic_examples
   select_academic_examples
   format_academic_examples
   top_venue_abstract_contract

4. 文件解析
   extract_pdf
   extract_docx / extract_doc
   extract_pptx / extract_ppt
   extract_image
   extract_tabular
   parse_uploaded_file

5. 本地兜底摘要
   select_key_sentences
   local_keywords
   local_academic_summary
   local_presentation_summary
   summarize_text

6. 大模型摘要
   get_llm_config
   call_llm_summary
   build_summary_prompt
   tools/prompt_templates.py 中的提示词模板
   summarize_text_with_llm
   summarize_chunks_with_llm

7. 摘要质量控制
   has_summary_pollution
   strip_summary_pollution
   is_low_quality_summary
   needs_top_venue_polish
   apply_pdca_summary_cycle
   apply_top_venue_final_review

8. 展示与导出
   render_route
   split_bilingual_summary
   render_summary
   render_analysis
   build_download_payload
```

## 摘要生成主流程

```text
上传文件
  -> parse_uploaded_file 提取文本
  -> analyze_text 分类与路由
  -> summarize_text_with_llm
       -> build_summary_prompt 生成初版摘要
       -> apply_pdca_summary_cycle 多轮审稿修复
       -> apply_top_venue_final_review 学术材料最终中英文审稿改写
       -> clean_summary_output 清洗污染内容
  -> render_summary 分中文/英文框展示
  -> save_document 保存历史记录
```

## 修改建议

- 新增文件格式：优先改 `parse_uploaded_file()` 和 `tools/file_router_config.json`。
- 新增分类：优先改 `STANDARD_CATEGORY_RULES` 和 `tools/file_router_config.json`。
- 调整摘要风格：优先改 `tools/prompt_templates.py`、`build_summary_prompt()`、`apply_top_venue_final_review()`。
- 调整质量复核：优先改 `is_low_quality_summary()`、`needs_top_venue_polish()`、`has_summary_pollution()`。
- 调整页面展示：优先改 `render_summary()`、`render_analysis()`。

## 后续可拆分方向

如果继续扩展，建议把 `app.py` 拆成以下模块：

```text
storage.py       用户、数据库、历史记录
extractors.py    PDF/Word/PPT/图片/表格解析
summarizers.py   本地摘要、大模型摘要、PDCA、顶刊审稿
quality.py       乱码、污染检测、摘要质量检查
ui.py            Streamlit 页面展示
```

当前没有直接拆分，是为了避免在功能频繁变化时引入导入路径和 Streamlit 状态问题。
