# 项目目录说明

本文档用于说明整理后的目录用途，方便交付、复现和后续维护。

## 核心运行文件

```text
app.py
requirements.txt
packages.txt
README.md
项目说明与运行部署.md
```

这些文件是运行和部署应用的核心文件。

## 运行数据

```text
data/app.db
data/uploads/
```

`data/app.db` 保存用户、文档记录和分析结果。  
`data/uploads/` 保存用户上传的原始文件。  

这两个路径是应用运行需要的，不建议随意删除。

## 训练和模型产物

```text
artifacts/models/
artifacts/sft/
```

`artifacts/models/` 保存训练好的模型参数包。  
`artifacts/sft/` 保存摘要微调训练集。  

这些文件不是 Streamlit 应用启动的必需文件，但用于后续摘要模型优化。

## 测试和实验材料

```text
experiments/quality_tests/
experiments/smoke_tests/
```

这里保存摘要质量测试、格式测试、长文测试和冒烟测试材料。  
正式演示时可以保留，也可以单独归档。

## 文档

```text
docs/
```

保存 OCR、摘要优化、文献采集、微调训练等说明文档。

## 工具脚本

```text
tools/
```

保存 OCR 检查、文献采集、PDF 下载、训练集构建等辅助脚本。

## 备份

```text
backup/
```

保存历史备份文件。当前包含 `app.py.bak_pdca`。

## Git 忽略规则

`.gitignore` 已忽略以下运行和产物目录：

```text
data/
artifacts/
experiments/
backup/
__pycache__/
*.pyc
.streamlit/secrets.toml
```

这样可以避免误提交用户上传文件、数据库、模型包、测试材料和真实 API Key。
