# 摘要算法优化记录

## 目标

本项目的摘要功能需要同时服务普通文档和论文/研究报告两类内容。优化目标不是简单缩短原文，而是生成更像人工摘要的结果：信息准确、结构清楚、语言自然、少重复、不编造事实，并在大模型不可用时仍能给出可读的本地摘要。

## 原有问题

1. 学术摘要范例库存在乱码内容，模型会把乱码当作风格参考，影响论文摘要质量。
2. 论文摘要提示词偏英文，且约束不够贴近中文论文摘要写法，容易出现“该文档提到”“本文属于某类文档”等转述口吻。
3. 长文直接分块摘要会多次调用模型，速度慢，失败概率高，且片段摘要容易丢失主线。
4. 大模型调用失败时系统静默回退到本地摘要，用户无法判断是 API 超时、密钥问题还是质量过滤失败。
5. 本地摘要算法主要依赖词频和关键句，重复句会被反复选中，导致摘要像关键词和原句拼接。
6. 英文或中英混排文档中，关键词容易被高频英文虚词或通用词占据，影响摘要开头质量。

## 已完成优化

### 1. 清理摘要范例库

文件：`academic_abstract_examples.jsonl`

已将乱码范例替换为正常中文范例，包括经验方法论文、系统应用论文和综述研究三类摘要范式。范例只作为风格参考，不作为事实来源。

同时在代码中增加 `looks_mojibake()`，加载范例时会跳过疑似乱码的摘要，避免异常数据继续污染提示词。

### 2. 强化论文摘要提示词

文件：`app.py`

`build_summary_prompt()` 中的学术摘要提示词已改为中文，明确要求：

- 只输出一段 300-500 字中文摘要
- 覆盖研究背景/问题、方法或系统设计、数据/实验/应用场景、主要结果、创新点或贡献、应用意义
- 避免“该文档提到”“本文属于某类文档”等转述口吻
- 不编造数据集、指标、年份、样本量或实验结论
- 原文缺少定量结果时只写定性发现

### 3. 优化普通文档摘要提示词

普通文档仍保留五个小标题：

- 核心概述
- 关键信息
- 重要结论
- 风险或注意事项
- 建议行动

提示词要求每个标题下写 1-2 句完整中文，并在没有风险或行动项时明确说明“原文未提出明确风险/行动项”，避免模型硬编内容。

### 4. 改进长文处理策略

新增 `build_llm_source()`：

- 普通长文优先取开头和结尾代表片段，减少无效分块调用。
- 学术长文优先抽取摘要、引言、结论等关键章节，再补充开头和结尾。
- 先基于代表片段生成正式摘要；如果质量检测不通过，再退回分块摘要。

这样可以减少等待时间，也更容易保留论文主线。

### 5. 改进本地回退摘要

本地摘要仍用于没有 API Key、API 失败或模型结果不合格时兜底。已优化：

- 增加句子签名 `sentence_signature()`，过滤重复句。
- 关键句选择时对高频词做限幅，避免某个重复词支配评分。
- 中文句子适当加权，减少中英混排文档中英文碎片支配摘要。
- 关键词优先选择中文词，中文不足时再使用英文词。

### 6. 显示大模型失败原因

`summarize_text_with_llm()` 不再静默吞掉异常。大模型失败后，摘要来源会显示类似：

```text
本地摘要（大模型调用失败：...）
```

这样验收时可以快速判断问题出在摘要算法、本地回退，还是 API 连接/超时。

## 当前摘要流程

1. 上传文件后提取文本。
2. 判断文本是否乱码或不可读。
3. 判断是否为论文/研究报告。
4. 如果配置了大模型 API：
   - 构造代表性输入片段。
   - 调用大模型生成摘要。
   - 对摘要做低质量检测。
   - 如质量不合格，调用二次润色提示词。
   - 长文必要时再进入分块摘要。
5. 如果未配置 API 或调用失败：
   - 使用本地关键句摘要。
   - 摘要来源中标明本地回退或失败原因。

## 质量验收标准

建议用以下标准检查摘要是否合格：

1. 不出现乱码、重复句、空泛套话。
2. 不出现“如果你需要”“我可以继续”等对话式内容。
3. 论文摘要应是一段正式学术中文，不应写成列表。
4. 普通文档摘要应保留关键事实、结论、风险和行动项。
5. 不编造原文没有的数字、结论、机构、日期或建议。
6. 长文摘要应覆盖全文主线，而不是只总结某一个片段。
7. 大模型不可用时，本地摘要应至少可读，不应大量重复。

## 后续可继续优化方向

1. 将摘要功能拆出 `summary.py`，避免 Streamlit UI 顶层执行影响自动化测试。
2. 为摘要模块增加测试样例，包括普通文档、论文、英文论文、扫描 OCR 文本和乱码文本。
3. 增加“摘要风格”选项，例如学术摘要、汇报摘要、会议纪要、办事清单。
4. 对论文类文档增加章节识别，分别抽取摘要、方法、实验、结果和结论。
5. 为 OCR 文本增加清洗规则，减少断行、页眉页脚和重复水印对摘要的影响。
6. 保存大模型失败日志，便于区分密钥错误、额度不足、网络超时和模型输出质量问题。
7. 如果后续有人工优质摘要数据，可以建立小型评测集，定期对比不同提示词和模型的摘要质量。

## 2026-07-01 多样化测试记录

已使用项目内置样例测试以下类型：

| 类型 | 样例 | 结果 |
| --- | --- | --- |
| 普通 Word | `data/format_smoke_test/sample_word.docx` | 大模型可生成结构化中文摘要，能识别原文缺少金额、日期和完整条款。 |
| 论文类 Word | `experiments/quality_tests/academic_example_library_test/paper_with_examples.docx` | 原始大模型输出质量合格，但接口偶发超时；已放宽学术摘要质量检测，避免合格摘要被固定关键词规则误杀。 |
| 长文 Word | `experiments/smoke_tests/long_text_smoke_test/long_10000.docx` | 本地摘要不再重复同一句，但长文样例文本质量较弱，仍建议优先使用大模型路径。 |
| 普通 PDF | `data/format_smoke_test/sample_pdf.pdf` | 可提取文本并生成摘要，但样例内容较短，摘要信息量受原文限制。 |
| 图片 OCR | `data/format_smoke_test/sample_image.png` | 本机未检测到 Tesseract，图片和扫描版 PDF 暂不能在本机完成 OCR 摘要。 |

本次测试后进一步修正了两点：

1. 本地摘要的概述、关键信息和结论不再重复使用同一批关键句，短文摘要更少重复。
2. 学术摘要质量检测从固定词匹配改为语义要素词组匹配，降低误判率。

## 2026-07-01 本地规则兜底增强

考虑到大模型 API 可能出现网络超时、额度不足或服务不可用，已进一步强化本地摘要路径。该路径不依赖外部网络，适合作为掉线兜底。

新增能力：

1. 英文高频技术词中文化，例如将 `document`、`ocr`、`pdf`、`word`、`cloud` 等映射为“文档”“OCR识别”“PDF解析”“Word解析”“云端部署”。
2. 论文/研究报告优先抽取摘要、方法、实验/结果、结论等章节，离线时也尽量保持学术摘要结构。
3. 普通文档本地摘要扩展为五段式，包含核心概述、关键信息、重要结论、风险或注意事项、建议行动。
4. 增加风险词和行动词抽取，例如“违约、缺少、风险、必须、建议、安装、部署”等。
5. 增加安全截断，避免长英文句被截成半个单词或半句话。
6. 继续保留句子去重和词频限幅，减少重复句对摘要的影响。

当前定位：

- 大模型可用时，仍以大模型摘要作为主路径。
- 大模型不可用时，本地规则摘要可以保证基本可读、结构完整、少重复。
- 本地规则不会真正理解和改写英文长句，因此英文论文的离线摘要仍会保留部分英文原句；如需完全中文化，仍建议恢复大模型调用或接入本地翻译/改写模型。

## 2026-07-01 中英文摘要测试记录

测试方式：

- 英文样例使用项目内置 Word/PDF 文件。
- 中文样例使用合同通知和论文摘要两段文本。
- 先强制关闭 API 测试本地规则兜底，再单独测试大模型在线路径。

测试结论：

1. 英文普通 Word 在线摘要效果较好，大模型能输出完整中文五段式摘要，并能指出原文缺少金额、日期和完整合同条款。
2. 英文论文离线摘要能够识别 OCR、PDF、Word、云端部署、用户隔离等主题，并抽取方法信息，但仍会保留部分英文原句。
3. 中文普通合同通知离线摘要表现较好，能识别合同/协议类别，抽取服务期限、合同金额、违约责任和 7 月 5 日前确认事项。
4. 中文论文摘要离线摘要能保持一段式学术摘要结构，覆盖问题、系统方法、实验结果和工程价值。
5. 本地规则摘要已修复末尾重复句号问题，避免拼接后出现“。。”。

当前判断：

- 中文文档：本地兜底已经基本可用。
- 英文文档：在线大模型摘要质量明显更好；离线规则摘要可以兜底，但不能完全替代大模型的中文改写能力。
