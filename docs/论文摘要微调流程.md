# 论文摘要 LoRA 微调流程

本流程用于把本地 PDF 论文整理成摘要训练样本，并在云端 GPU 机器上微调一个论文摘要 LoRA 适配器。

## 1. 生成训练数据

先用已有 PDF 跑通小样：

```powershell
cd /d D:\DESTOP\老师的题目_V2
python tools\build_paper_sft_dataset.py `
  --pdf-dir "H:\transformer对破产影响记录\100篇顶会顶刊Transformer论文&代码合集" `
  --out artifacts\sft\paper_summary_sft.jsonl `
  --limit 20
```

如果已配置 `LLM_API_KEY`、`LLM_API_BASE`、`LLM_MODEL`，可以生成更高质量中文摘要标签：

```powershell
$env:LLM_API_KEY="your_api_key"
$env:LLM_API_BASE="https://api.tokensmart.ai/v1"
$env:LLM_MODEL="gpt-5.4-mini"
python tools\build_paper_sft_dataset.py `
  --pdf-dir "H:\transformer对破产影响记录\100篇顶会顶刊Transformer论文&代码合集" `
  --out artifacts\sft\paper_summary_sft.jsonl `
  --use-llm
```

输出文件是 chat SFT 格式，每行一个样本。

## 2. 上传到云端

建议放到云端：

```text
/root/doc_summary_lora/data/paper_summary_sft.jsonl
/root/doc_summary_lora/train_lora_cloud.sh
```

## 3. 云端训练

云端机器有 RTX 5090，适合先做 7B/8B 模型 LoRA。

```bash
cd /root/doc_summary_lora
bash train_lora_cloud.sh
```

默认使用：

```text
Qwen/Qwen2.5-7B-Instruct
```

输出目录：

```text
/root/doc_summary_lora/outputs/qwen2_5_7b_paper_summary_lora
```

## 4. 注意事项

- 不要直接用 PDF 文件训练，必须先转成“论文内容 -> 高质量摘要”的样本。
- 只有自动弱监督摘要时，微调效果有限；最好用大模型或人工校对生成目标摘要。
- 几百篇论文适合做 LoRA 风格微调，不适合从头训练模型。
- 如果要覆盖所有领域，应继续扩充中英文论文样本，并保持每个领域摘要质量一致。
