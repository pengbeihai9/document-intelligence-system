# 大模型摘要稳定性说明

## 已解决的问题

大模型 API 偶发超时或返回空内容时，系统不再立即回退本地摘要，而是先执行：

1. 当前模型重试。
2. 如果配置了备用模型，切换备用模型继续重试。
3. 所有模型都失败后，才回退本地摘要。

摘要来源会显示实际成功的模型，例如：

```text
大模型学术摘要（gpt-5.4-mini）
```

如果全部失败，则显示：

```text
本地摘要（大模型调用失败：...）
```

## Streamlit Secrets 配置

```toml
LLM_API_KEY = "your_api_key"
LLM_MODEL = "gpt-5.4-mini"
LLM_BACKUP_MODELS = "gpt-5.4-mini,gpt-5.4"
LLM_API_BASE = "https://api.tokensmart.ai/v1"
```

说明：

- `LLM_MODEL` 是主模型。
- `LLM_BACKUP_MODELS` 是备用模型列表，使用英文逗号、分号或换行分隔。
- 如果备用列表里包含主模型，系统会自动去重。
- 每个模型默认最多尝试 2 次。

## 本地兜底

如果所有模型都失败，系统仍会自动使用本地规则摘要。中文文档本地摘要较稳；英文文献本地摘要可提取核心信息，但中文改写质量不如大模型。
