# 文档智能处理系统交付说明

## 项目简介

本项目实现了一个文档智能处理 Web 小工具。系统支持用户注册登录，登录用户可以上传 PDF、Word、图片或扫描版 PDF 文件。系统会自动提取文档文本，对扫描件执行 OCR 识别，并生成摘要、词频统计图、词云和文档分类结果。

系统使用 SQLite 保存用户和文档记录，每条文档记录都绑定上传用户。用户登录后只能在“我的文档”中查看自己上传过的文件记录。

## 已实现要求

| 要求 | 实现情况 |
| --- | --- |
| 多用户注册登录 | 已实现，支持注册、登录、退出 |
| 用户文件隔离 | 已实现，按用户过滤历史文档 |
| PDF 上传并提取文本 | 已实现，使用 PyMuPDF |
| Word 上传并提取文本 | 已实现，支持 `.docx`；`.doc` 会尝试通过 Microsoft Word 或 LibreOffice 自动转换 |
| 图片 OCR | 已实现，使用 Tesseract OCR |
| 扫描版 PDF OCR | 已实现，PDF 无文本时转图片 OCR |
| 文本总结 | 已实现，基于词频和句子权重生成摘要 |
| 词频统计/可视化 | 已实现，包含柱状图和词云 |
| 文件分类 | 已实现，基于关键词规则自动分类 |
| 公网可访问 | 可部署到 Streamlit Community Cloud 获取公网链接 |

## 自动分类逻辑

系统内置五类常见文档规则：

- 合同/协议
- 财务/票据
- 简历/人才材料
- 论文/研究报告
- 通知/公文

程序会统计不同类别关键词在全文中的出现次数，选择得分最高的类别作为分类结果。如果所有类别得分均为 0，则分类为“其他文档”。

## 运行方式

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 部署方式

1. 将项目上传到 GitHub 仓库。
2. 打开 Streamlit Community Cloud：https://share.streamlit.io/
3. 选择 GitHub 仓库。
4. 入口文件填写 `app.py`。
5. 点击部署，得到公网访问链接。
