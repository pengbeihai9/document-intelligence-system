# 文档智能处理系统 V2：项目说明与运行部署

## 一、项目简介

本项目是一个文档智能处理 Web 小工具第二版，支持用户注册登录，并优化了连续上传体验。登录后，用户可以上传 PDF、Word、图片和扫描版 PDF 文件，系统会自动提取文本，对扫描件执行 OCR 识别，并生成文本摘要、词频统计可视化和文档类别判断结果。

系统会保存每个用户的上传记录，用户只能在自己的账号下查看自己上传过的文档历史。

## 六、使用方法

1. 打开系统网页。
2. 注册一个新账号，或使用已有账号登录。
3. 上传 PDF、Word 或图片文件。
4. 系统自动解析文档内容。
5. 如果是扫描件或图片，系统自动 OCR 识别。
6. 页面展示自动分类、文本摘要、词频柱状图、词云和提取全文。
7. 侧边栏“我的文档”只显示当前登录用户上传过的文档。
8. 可点击“下载提取文本”保存识别结果。

## 七、自动分类逻辑

系统内置五类文档规则：

- 合同/协议
- 财务/票据
- 简历/人才材料
- 论文/研究报告
- 通知/公文

程序会统计各类别关键词在全文中出现的次数，选择得分最高的类别作为分类结果。如果所有类别得分均为 0，则归为“其他文档”。

## 八、公网部署方法

可部署到 Streamlit Community Cloud：

部署上线后，用户只需要打开公网链接使用。文档解析、图片 OCR、扫描版 PDF OCR、摘要、词频统计和分类都在云端服务器执行，访问用户的电脑不需要安装 Python、Tesseract、Poppler 或任何依赖。

1. 打开 GitHub，新建一个仓库。
2. 上传本项目目录中的文件。
3. 打开 Streamlit Cloud：

```text
https://share.streamlit.io/
```

4. 使用 GitHub 登录。
5. 点击 New app。
6. 选择刚才上传的 GitHub 仓库。
7. Main file path 填写：

```text
app.py
```

8. 点击 Deploy。
9. 部署成功后会得到公网访问链接，例如：

```text
https://你的项目名.streamlit.app
```

## 十、注意事项

本地运行时，如果电脑没有安装 Tesseract OCR，图片和扫描件 OCR 可能无法正常识别，但普通 PDF 和 Word 文档通常可以正常解析。

云端部署时，`packages.txt` 已经配置 OCR 所需依赖：

```text
tesseract-ocr
tesseract-ocr-chi-sim
poppler-utils
```

部署平台会根据该文件自动安装相关系统依赖。


## 十一、大模型智能摘要配置

V2 支持接入 OpenAI 兼容接口生成更智能的摘要。配置 `LLM_API_KEY` 后，系统会优先调用大模型；如果没有配置或调用失败，会自动回退到本地摘要算法。

Streamlit Cloud 的 Secrets 示例：

```toml
LLM_API_KEY = "pk_live_your_api_key"
LLM_MODEL = "gpt-5.4-mini"
LLM_API_BASE = "https://api.tokensmart.ai/v1"
```

不要将真实 API Key 写入代码或提交到 GitHub。
